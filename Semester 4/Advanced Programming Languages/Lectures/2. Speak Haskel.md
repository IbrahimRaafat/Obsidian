# Haskell: Core Concepts and Functions Summary

## 1. Fundamental Principles

- **Declarative Nature**: Instead of writing instructions (imperative), you write **definitions** that describe what things _are_.
    
- **Referential Transparency**: A function always returns the same result for the same input, with no hidden side effects.
    
- **Lazy Evaluation**: Expressions are only computed when their values are actually needed.
    
- **Strict Typing**: Haskell is strongly and statically typed. It uses **Type Inference** to deduce types, though explicit declarations are encouraged for clarity.
    

---

## 2. Functions and Application

- **Function Calls**: Invoked using spaces rather than parentheses and commas.
    
    - _Usage_: `max 3 7` or `min 10 5`.
        
- **Type Signatures**: Written as `name :: InputType -> OutputType`.
    
    - _Example_: `double :: Int -> Int`.
        
- **Partial Application**: Functions do not need all arguments at once. Providing some arguments returns a new function.
    
    - _Example_: `addFive = add 5` creates a function that adds 5 to its input.
        
- **Anonymous Functions (Lambdas)**: Functions without a name, written as `\argument -> expression`.
    
    - _Example_: `(\x -> x * x + 1)`.
        

---

## 3. Collections: Lists and Tuples

### Lists

Lists are homogeneous (all elements must be the same type) and variable-length.

- **Structure**: A list is either empty `[]` or a head followed by a tail `x:xs`.
    
- **Common Functions**:
    
    - `head`: Returns the first element.
        
    - `tail`: Returns everything but the first element.
        
    - `last`: Returns the last element.
        
    - `init`: Returns everything but the last element.
        
    - `length`: Returns the number of elements.
        
    - `reverse`: Reverses the list.
        
    - `null`: Checks if the list is empty.
        

### Tuples

Tuples are heterogeneous (can hold different types) but have a fixed length.

- **Selectors**: `fst` and `snd` work only for pairs (2-tuples).
    
- **Usage**: `("Alice", 30) :: (String, Int)`.
    

---

## 4. Recursion and Pattern Matching

- **Recursion**: Repeating actions by having a function call itself, typically requiring a base case and a recursive step.
    
- **Pattern Matching**: Defining functions by specifying patterns that the data must match.
    
    - _List Recursion Example_: `sum' [] = 0` (base case) and `sum' (x:xs) = x + sum' xs` (recursive step).
        
    - _Wildcard (`_`)_: Used in patterns when you do not need the value of a specific element.
        

---

## 5. List Comprehensions

A concise way to build lists using the syntax `[expression | generator, guard]`.

- **Generator**: Defines where values come from (`x <- [1..10]`).
    
- **Guard**: Filters values based on a condition (`even x`).
    
- _Example_: `[x^2 | x <- [1..10], even x]` produces squares of even numbers from 1 to 10.
    

---

## 6. Higher-Order Functions and Composition

Higher-order functions take other functions as arguments or return them as results.

- **`map`**: Applies a function to every element in a list.
    
    - _Usage_: `map double [1, 2, 3]` results in `[2, 4, 6]`.
        
- **Function Composition (`.`)**: Chains functions together (executes right-to-left).
    
    - _Example_: `(reverse . map toUpper) "hello"` results in `"OLLEH"`.
        
- **Function Application (`$`)**: Used to avoid nested parentheses by evaluating everything to its right first.
    

---

## 7. Advanced Concepts

- **Parametric Polymorphism**: Functions that work identically for any type, represented by type variables (e.g., `a`).
    
- **Type Classes**: Constraints that restrict polymorphic types to those supporting certain operations.
    
    - _Example_: `Num a =>` ensures the type `a` supports numeric operations.
        
- **Monads**: Constructs used to handle side effects like I/O explicitly within the type system (e.g., `IO` monad)
